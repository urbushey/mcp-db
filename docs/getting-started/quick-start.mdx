## Prerequisites

- [Bun](https://bun.sh) v1.0 or later
- [Claude Desktop](https://claude.ai/download)

---

## Install

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/your-username/instant-db
    cd instant-db
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    bun install
    ```
  </Step>

  <Step title="Copy the environment file">
    ```bash
    cp .env.example .env
    ```

    The defaults work for most setups. Edit `.env` if you want to store data or logs somewhere specific.

    ```bash .env
    DATA_DIR=./data           # Where .sqlite files are stored
    LOG_LEVEL=normal          # verbose | normal | off
    LOG_PATH=./logs/mcp.log   # Log file location
    MCP_TRANSPORT=stdio       # Always stdio for Claude Desktop
    ```
  </Step>

  <Step title="Connect to Claude Desktop">
    See the [Claude Desktop Setup](/getting-started/claude-desktop-setup) page for the exact config snippet to add.
  </Step>
</Steps>

---

## Verify the server starts

Before adding it to Claude Desktop, confirm the server runs without errors:

```bash
bun run src/index.ts
```

The server waits for MCP messages over stdio. You won't see output in normal mode — that's expected. Press `Ctrl+C` to stop it.

---

## Run the tests

```bash
bun test
```

You should see 50 tests pass across 9 files.

---

## Your first database

Once connected to Claude Desktop, open a new conversation and say:

> "I want to track my daily water intake — date, glasses of water, and any notes."

Claude will:
1. Propose a table structure conversationally and wait for your confirmation
2. Call `create_database` to create it
3. Be ready to insert and query records immediately

<Tip>
  Add the [recommended system prompt](/getting-started/claude-desktop-setup#system-prompt) to Claude so it automatically checks for existing databases at the start of each session.
</Tip>
