instant-db supports four column types. They map directly to SQLite's type affinity system.

---

## Types

| instant-db type | SQLite affinity | Use for |
|---|---|---|
| `text` | TEXT | Names, descriptions, dates (as strings), free text |
| `integer` | INTEGER | Counts, IDs, whole numbers, booleans (see below) |
| `real` | REAL | Weights, prices, measurements, any decimal value |
| `boolean` | INTEGER (0/1) | Flags, yes/no fields |

<Note>
  SQLite stores booleans as integers (0 = false, 1 = true). When Claude inserts a boolean value, it is converted automatically.
</Note>

---

## Dates and times

There is no native `date` or `timestamp` type. The recommended approach is `text` with [ISO 8601](https://en.wikipedia.org/wiki/ISO_8601) format strings (`YYYY-MM-DD` or `YYYY-MM-DDTHH:mm:ssZ`).

ISO 8601 strings sort correctly as text, so `orderBy` on a date column works as expected.

```
Good:  "2024-01-15"
Good:  "2024-01-15T08:30:00Z"
Avoid: "January 15, 2024"   ← doesn't sort correctly
Avoid: 1705305600           ← Unix timestamp, harder to read in queries
```

---

## The `id` column

Every table gets an `id INTEGER PRIMARY KEY AUTOINCREMENT` column automatically, unless you explicitly define a column with `primaryKey: true` in your schema. You never need to include `id` when inserting records.

---

## Column definition reference

```typescript
interface ColumnDef {
  name: string;
  type: "text" | "integer" | "real" | "boolean";
  required?: boolean;   // Adds NOT NULL constraint
  primaryKey?: boolean; // Marks as PRIMARY KEY AUTOINCREMENT
}
```
